LCA的求法：

（1）先朴素的考虑这个问题，得出暴力做法：向上标记法，先$ dfs/bfs $求一遍每个节点的深度，对于任意两个节点，当求解它们的$ LCA $时每次向上走一格，先让他们向上走到相同高度，然后继续向走跳直到第一次相遇，所得即为$ LCA $ 。复杂度为$ O(n) $。

（2）考虑到一次只跳一格导致复杂度过高，使用倍增优化：我们定义$ f_{i, j} $表示从$ i $节点开始，向上走$ 2^{j} $ ${(0 <= j <= log{2}n}) $步能走到的节点，$ depth[i] $记录$ i $号节点的深度，这里需要加一个特判，当从$ i $开始跳$ 2^{j} $步会跳过根节点的时候，令$ f_{i, j} = 0, depth[0] = 0 $。这样可以做到$ O(nlogn) $的预处理，实现$ O(logn) $的查询。具体步骤第二步与方法一有些差异，让两个点同时往上跳的时候跳到它们的最近公共祖先下一层，这时答案即为$ f_{a,0}(或f_{b, 0}) $。
